# T4 检索接口规约
# 课题四 × T4 算法组接口对接文档

> 当前结论：T4 暂不需要算法组实现独立 HTTP 接口。文本/图像 embedding 生成，以及“文搜图”检索，都由后端内部实现。

---

## 一、当前职责划分

| 能力 | 当前实现方 | 说明 |
|---|---|---|
| 文本 embedding 生成 | 后端 `EmbeddingService` | 直接调用 `llm.embedding.*` 配置指向的 embedding 服务 |
| 图像 embedding 生成 | 后端 `EmbeddingService` | 直接调用同一个 embedding 服务 |
| 文本内容向量入库 | 后端 | `T4IndexingStep` 生成标题/正文向量并写入 Milvus `media_content_embeddings` |
| 媒体资源向量入库 | 后端 | `ImageEmbeddingService` 生成视觉/OCR/ASR/caption 向量并写入 Milvus `media_asset_embeddings` |
| 实体/账号/内容节点向量入库 | 后端 | 后端生成向量并统一写入 `entity_embeddings` |
| 文搜图/视频 | 后端 | 用户文本生成 query embedding 后，后端直接查 Milvus `media_asset_embeddings` 的视觉/OCR/ASR/caption 向量 |

也就是说，T4 不再通过 `AgentProxyClient.call("T4", "generate_text_embedding", ...)` 或 `generate_image_embedding` 这类 action 调用。embedding 生成是后端基础设施能力，不作为算法组接口交付。

---

## 二、后端内部 embedding 能力

后端新增内部服务：

```java
com.idata.profile.infra.embedding.EmbeddingService
```

核心方法：

```java
float[] generateTextEmbedding(String text);
float[] generateImageEmbedding(String imageUrl);
```

配置来源：

```yaml
llm:
  embedding:
    base-url: http://172.16.40.232:18013/v1
    api-key: idata@123456
    model: Qwen3-VL-Embedding-8B
```

调用方式：后端直接请求 `${llm.embedding.base-url}/embeddings`，请求体形如：

```json
{
  "model": "Qwen3-VL-Embedding-8B",
  "input": "文本或图片URL"
}
```

返回的 embedding 维度必须与 Milvus collection 的 `embedding` 字段一致，当前为 4096。

## 三、文搜图当前实现

“智能融合”里选择 `targetModalities=image` 时，后端执行：

```text
用户文本 query
  -> EmbeddingService.generateTextEmbedding(query)
  -> MilvusVectorService.searchMediaAssetTextEmbeddingsWithScore(...)
  -> 得到 media_assets.id + similarity score
  -> 后端查询 media_assets / media_contents 补齐图片URL和所属贴文信息
  -> 前端以图片网格展示，点击图片进入所属贴文详情
```

Milvus 查询范围只包含：

```text
Collection: media_asset_embeddings
Vector fields: visual_embedding / ocr_embedding / asr_embedding / caption_embedding
Output fields: asset_id / segment_id
```

其中 `asset_id` 对应 PostgreSQL `media_assets.id`。

## 四、接口状态

以下 action 不再作为 T4 算法接口要求：

| action | 状态 | 替代方案 |
|---|---|---|
| `generate_text_embedding` | 不再要求算法组实现 | 后端 `EmbeddingService.generateTextEmbedding` |
| `generate_image_embedding` | 不再要求算法组实现 | 后端 `EmbeddingService.generateImageEmbedding` |
| `search_images_by_text` | 暂不要求算法组实现 | 后端直接生成 query embedding 并查 Milvus `media_asset_embeddings` |

如果后续算法组要在“文搜图”上增加额外重排序、视觉语义解释、跨模态校准等能力，再单独定义新的 T4 检索接口；在当前阶段不需要。

---

## 五、边界原则

1. 后端负责 embedding 生成、Milvus 写入、Milvus 基础向量召回。
2. 后端负责根据 `assetId` 查询 PG，补齐图片 URL、所属贴文、平台、正文摘要等展示字段。
3. 算法组不需要访问 PostgreSQL、Elasticsearch、Neo4j。
4. 算法组当前不需要实现 T4 HTTP action。
5. 如果未来新增 T4 算法检索接口，响应也应优先返回轻量候选 ID 和分数，业务字段由后端补齐。
