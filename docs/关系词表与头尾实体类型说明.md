# 关系词表与头尾实体类型说明

本文档用于同步当前 RZDK 中所有关系类型的统一口径，面向所有共同开发者。

当前关系定义的代码来源：
- `RZDK/src/core/models/relationships.py`
- `RZDK/docs/T2_指标1.2_10类实体6类关系设计说明.md`

## 关系总览

当前系统以 `RZDK/src/core/models/relationships.py` 中的 `RelationType` 枚举为唯一准入词表，**共 43 个关系类型**。本文档已经按现有 7 个分组把这 43 个关系全部展开说明，不存在“只列核心关系、未列全量关系”的情况。

7 个分组及数量如下：

1. 身份归一类：4 个
2. 组织/社群类：7 个
3. 内容链路类：8 个
4. 事件地点类：6 个
5. 叙事认知类：9 个
6. 传播协同类：5 个
7. 证据治理类：4 个

其中需要明确区分两层口径：

- **代码已强约束的 7 类核心关系**：`INFLUENCES`、`SUPPORTS`、`OPPOSES`、`INCITES`、`DE_ESCALATES`、`BELONGS_TO`、`CONTROLS`
- **当前团队约定口径的其他 36 类关系**：已经进入统一词表，但在后端代码中尚未建立同等严格的头尾实体类型校验；本文档给出的头尾类型属于当前推荐实现口径

## 范围说明

当前系统中的关系词表以 `RelationType` 枚举为准，共分为 7 组：

1. 身份归一类
2. 组织/社群类
3. 内容链路类
4. 事件地点类
5. 叙事认知类
6. 传播协同类
7. 证据治理类

其中：
- `影响 / 支持 / 反对 / 煽动 / 平息 / 隶属 / 控制` 这 7 类核心关系，已经在 `relationships.py` 中有正式的头尾实体类型约束
- 除上述 7 类以外，其余 36 类关系目前**只有统一关系名词表**，尚未在后端模型中形成同等严格的头尾类型校验，因此本文档对其给出的头尾实体类型属于**当前推荐口径**

## 当前 6 个基础实体类

本文档中的实体类型统一使用当前主口径的 6 个基础实体类：

- `Person`
- `Organization`
- `SocialAccount`
- `MediaContent`
- `Event`
- `Location`

补充说明：
- 当前 `entities_new.py` 中不再把 `Narrative` 作为 T2 原子实体
- 历史关系名中仍有 `NARRATIVE_*`、`CONTENT_EXPRESSES_NARRATIVE` 等名称，当前可以保留关系名不变，但在前端展示或后续抽取实现中，应逐步向 6 基础实体类口径收敛

## 1. 身份归一类

### `SAME_AS`

- 中文关系名：同一实体 / 同一账号归一
- 解释：表示两个节点本质上指向同一个真实对象，常用于跨平台账号归一、重复实体归并前的同一性判断
- 推荐头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`、`Location`
- 推荐尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`、`Location`

### `HAS_ACCOUNT`

- 中文关系名：拥有账号 / 关联账号
- 解释：表示现实主体拥有、使用或稳定关联某个线上账号节点
- 推荐头实体类型：`Person`、`Organization`
- 推荐尾实体类型：`SocialAccount`

### `ALIAS_OF`

- 中文关系名：别名属于 / 别名指向
- 解释：表示一个名字、历史名称或别称指向另一个规范实体
- 推荐头实体类型：`Person`、`Organization`、`Location`
- 推荐尾实体类型：`Person`、`Organization`、`Location`

### `MERGED_INTO`

- 中文关系名：并入 / 归并到
- 解释：表示一个节点在去重或事件归并后，被合并进另一个节点
- 推荐头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`、`Location`
- 推荐尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`、`Location`

---

## 2. 组织/社群类

### `AFFILIATED_WITH`

- 中文关系名：关联 / 任职 / 隶属机构
- 解释：表示人物与机构、机构与机构之间存在稳定的任职、归属、挂靠或体系关联
- 推荐头实体类型：`Person`、`Organization`
- 推荐尾实体类型：`Organization`

### `PART_OF`

- 中文关系名：属于组成部分 / 上下级组成关系
- 解释：表示一个组织是另一个组织的组成部分、下级单位或内部结构单元
- 推荐头实体类型：`Organization`
- 推荐尾实体类型：`Organization`

### `CONTROLS`

- 中文关系名：控制
- 解释：表示主体对另一个主体或事件存在管理、运营、指挥、操盘或资助支配关系
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`
- 正式尾实体类型：`Organization`、`SocialAccount`、`Event`

### `OWNS`

- 中文关系名：拥有 / 持有
- 解释：表示主体拥有某个内容资产、媒体资产或平台载体
- 推荐头实体类型：`Person`、`Organization`
- 推荐尾实体类型：`MediaContent`、`SocialAccount`

### `MEMBER_OF`

- 中文关系名：成员属于
- 解释：表示人物是某组织、社群或体系中的成员
- 推荐头实体类型：`Person`、`SocialAccount`
- 推荐尾实体类型：`Organization`、`SocialAccount`

### `ADMIN_OF`

- 中文关系名：管理 / 管理员属于
- 解释：表示人物或账号对某个账号、频道、社群具有管理职责
- 推荐头实体类型：`Person`、`SocialAccount`
- 推荐尾实体类型：`SocialAccount`

### `PUBLISHED_IN`

- 中文关系名：发布于 / 刊载于
- 解释：表示内容被发布在某机构化媒体、栏目或发布容器中
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`Organization`、`SocialAccount`

---

## 3. 内容链路类

### `REPLY_TO`

### `COMMENT_ON`

- 解释：表示一条内容是对另一条内容的评论
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `REPOSTS`

- 中文关系名：转发
- 解释：表示一条内容转发了另一条内容
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `QUOTES`

- 解释：表示一条内容引用了另一条内容并附带自身表达
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `SHARES`

- 中文关系名：分享
- 解释：表示一条内容将另一条内容或其链接进行再次传播
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `REFERENCES_URL`

- 中文关系名：引用链接
- 解释：表示一条内容引用、指向或附带某个外部 URL 资源
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `MENTIONS`

- 解释：表示一条内容中提及某个实体
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`Person`、`Organization`、`SocialAccount`、`Event`、`Location`

## 5. 叙事认知类

> 说明：这一组保留的是历史关系名。由于当前 T2 设计中不再将 `Narrative` 作为原子实体，后续实现应逐步把这些关系向 `Person / Organization / SocialAccount / MediaContent / Event / Location` 六类基础实体收敛。

### `CONTENT_EXPRESSES_NARRATIVE`

- 中文关系名：内容表达叙事
- 解释：表示某条内容承载、表达或体现某类叙事、主张或框架
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`Event`、`Person`、`Organization`

### `NARRATIVE_TARGETS_ENTITY`

- 中文关系名：叙事指向实体
- 解释：表示某类叙事、主张或框架主要针对某个实体对象
- 推荐头实体类型：`MediaContent`、`Event`
- 推荐尾实体类型：`Person`、`Organization`、`SocialAccount`

### `NARRATIVE_ABOUT_EVENT`

- 中文关系名：叙事关于事件
- 解释：表示某类叙事、主张或框架围绕某个事件展开
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`Event`

### `SUPPORTS`

- 中文关系名：支持
- 解释：表示主体 A 对主体 B 或事件 B 表达正向立场、认可、声援、背书、帮助或维护
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`
- 正式尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`

### `OPPOSES`

- 中文关系名：反对
- 解释：表示主体 A 对主体 B 或事件 B 表达负向立场、批评、质疑、否定、谴责、抵制或驳斥
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`
- 正式尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`

### `INCITES`

- 中文关系名：煽动
- 解释：表示主体 A 对事件 B、群体 B 或传播过程 B 进行动员、鼓动、扩散、放大、激化或推动传播
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`
- 正式尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`

### `DE_ESCALATES`

- 中文关系名：平息
- 解释：表示主体 A 对冲突 B、争议 B、恐慌 B 或升级风险 B 进行降温、澄清、调停、事实核查、缓和或阻断扩散
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`
- 正式尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`

### `BELONGS_TO`

- 中文关系名：隶属
- 解释：表示主体 A 与主体 B 之间存在较稳定的归属、任职、成员、挂靠、账号归口或组织从属关系
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`
- 正式尾实体类型：`Person`、`Organization`、`SocialAccount`

### `HAS_EMOTION`

- 中文关系名：具有情绪 / 情绪指向
- 解释：表示某条内容承载、体现或表达某种情绪倾向
- 推荐头实体类型：`MediaContent`
- 推荐尾实体类型：`Event`、`Person`、`Organization`

## 6. 传播协同类

### `AMPLIFIES`

- 中文关系名：放大 / 扩散放大
- 解释：表示某主体对某个内容、表达或传播过程进行了放大和二次传播
- 推荐头实体类型：`SocialAccount`、`Organization`
- 推荐尾实体类型：`MediaContent`、`Event`

### `BRIDGES_COMMUNITY`

- 中文关系名：跨社群桥接
- 解释：表示某主体在两个传播社区、账号簇或讨论圈层之间起桥接作用
- 推荐头实体类型：`SocialAccount`
- 推荐尾实体类型：`SocialAccount`、`Organization`

### `COORDINATES_WITH`

- 中文关系名：协同 / 协调
- 解释：表示两个主体之间存在协同、联动、共同行动或协调关系
- 推荐头实体类型：`Person`、`Organization`、`SocialAccount`
- 推荐尾实体类型：`Person`、`Organization`、`SocialAccount`

### `POTENTIAL_SUBORDINATE_TO`

- 中文关系名：疑似从属
- 解释：表示两个主体之间存在待确认的潜在从属关系，通常是弱证据、候选态或推断态
- 推荐头实体类型：`Person`、`Organization`、`SocialAccount`
- 推荐尾实体类型：`Organization`、`SocialAccount`

### `INFLUENCES`

- 中文关系名：影响
- 解释：表示主体 A 对主体 B 或事件 B 的观点、态度、判断、行为或传播方向产生影响、引导、塑造或推动作用
- 正式头实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`
- 正式尾实体类型：`Person`、`Organization`、`SocialAccount`、`MediaContent`、`Event`

## 7. 证据治理类

### `ASSERTED_BY`

- 中文关系名：由…主张 / 由…断言
- 解释：表示某个结论、叙事、事件判断或结构化关系是由某条内容或某个来源明确主张出来的
- 推荐头实体类型：`Event`、`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `DERIVED_FROM`

- 中文关系名：派生自
- 解释：表示某个结构化结论、关系或摘要是从某条内容、证据或上游事实派生出来的
- 推荐头实体类型：`Event`、`MediaContent`
- 推荐尾实体类型：`MediaContent`

### `CONFLICTS_WITH`

- 中文关系名：与…冲突
- 解释：表示两个内容、两个结论或两个事实主张之间存在明显冲突
- 推荐头实体类型：`MediaContent`、`Event`
- 推荐尾实体类型：`MediaContent`、`Event`

### `REVIEWED_BY`

- 中文关系名：由…复核
- 解释：表示某条内容、某个结论或某个结构化事实经过某人物或复核主体审核
- 推荐头实体类型：`MediaContent`、`Event`
- 推荐尾实体类型：`Person`、`Organization`

## 开发注意事项

1. 后端关系词表以 `RelationType` 为唯一准入词表，不要再新增自由文本关系名。

2. 当前只有 7 类核心关系具备正式的代码级头尾约束：
   - `INFLUENCES`
   - `SUPPORTS`
   - `OPPOSES`
   - `INCITES`
   - `DE_ESCALATES`
   - `BELONGS_TO`
   - `CONTROLS`

3. 其余关系当前文档给出的头尾实体类型是推荐实现口径；如果后续要写入严格校验，请优先补齐 `relationships.py` 中的正式约束映射。

4. 当前 T2 设计主口径是 6 个基础实体类，因此涉及历史 `Narrative` 命名的关系，后续应优先通过 `MediaContent / Event / Person / Organization / SocialAccount` 的组合方式落地，不建议再把 `Narrative` 恢复为 T2 原子实体。
