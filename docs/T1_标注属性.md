# T1 标注属性（讨论修订稿）

## 一、文档目的

本文件用于讨论 T1 模块对**单条采集内容**的自动化标注维度。这里的“单条采集内容”通常指：

- 一条社交媒体贴文
- 一条评论
- 一条回复
- 一篇新闻报道
- 一条转发 / 引用内容

任务书中明确要求该步骤要“进行事件热度、账户类别、核心立场等不少于 8 个维度的自动化标注”，因此这些维度必须保留。

但同时，T1 的输入粒度决定了：它更适合做**单条内容级、表达级、风险级**的判断，而不适合在这一层直接输出需要跨内容聚合才能成立的结论。

因此，本次修订建议采用两层结构：

1. **保留并优化的 T1 标注维度**
2. **建议删除或下沉到后续模块的维度**

---

## 二、总体修订原则

### 1. 只保留单条内容可判断的维度

如果一个结论必须结合账号历史发言、评论区整体分布、回复链传播结构或多条内容才能成立，就不应强行放在 T1。

### 2. 保留任务书中已经框死的核心维度

尤其是：
- 事件热度
- 账户类别
- 内容级核心立场

这些不能删除，只能优化定义与使用边界。

### 3. 把“内容判断”和“对象判断”分开

T1 更适合判断：
- 这条内容在表达什么
- 这条内容情绪如何
- 这条内容在做解释、宣传还是动员
- 这条内容是否带有操纵性表达

T1 不适合直接判断：
- 某个实体对另一个实体的稳定立场
- 民众整体态度
- 账号最终是什么类型的重点目标

### 4. 增加对 T6 有直接帮助的标注

新增 BEND 叙事操纵手法标注，用于后续重点目标识别。

---

## 三、建议保留并优化的 10 个标注维度

以下是建议保留的 T1 标注维度，采用讨论用中文表述。

### 1. 意识形态

判断内容体现的政治倾向、价值倾向和意识形态取向。

说明：
- 这里标注的是**这条内容表达出来的意识形态倾向**，不是发帖人长期稳定意识形态身份。
- 如果文本没有明显意识形态色彩，应允许标为“弱相关 / 不明显 / 无法判断”。

建议判定关注点：
- 是否明显偏向民族主义、保守主义、自由主义、宗教立场、反建制立场等
- 是否带有明显价值判断框架
- 是否在用某种意识形态解释事件

建议输出形式：
- 倾向标签
- 强度
- 证据片段

### 2. 内容级核心立场

判断整条内容总体是支持、反对、中立还是立场混合。

说明：
- 这是任务书明确要求保留的核心维度。
- 这里判断的是**整条内容的总体立场方向**，不是“谁对谁”的精细关系。
- 该维度适合 T1，因为它是内容整体表达层面的判断。

建议标签：
- 支持
- 反对
- 中立
- 混合
- 无法判断

### 3. 观点情绪

判断内容的情感极性、主要情绪和情绪强度。

说明：
- 这是内容表达特征，不是对象级关系。
- 适合直接从单条内容判断。

建议拆分为三部分：
- 情绪极性：正向 / 负向 / 中性 / 混合
- 主要情绪：愤怒 / 恐惧 / 悲伤 / 喜悦 / 自豪 / 焦虑 / 冷静等
- 情绪强度：低 / 中 / 高

### 4. 事件热度

判断事件的关注程度、传播热度和爆发趋势。

说明：
- 这是任务书明确要求保留的维度。
- 但要注意：T1 对“事件热度”的判断应是**基于当前这条内容所体现出的热度信号**，而不是基于全网统计结果。
- 它更像“热度感知标注”，不是最终传播量结论。

建议判定依据：
- 是否使用“刷屏”“全网关注”“持续发酵”“热搜”“引爆讨论”等热度措辞
- 是否表现出明显的事件升级感、爆发感、集中关注感
- 是否带有“刚刚发生 / 正在发酵 / 持续升温”的时间态势

建议标签：
- 低
- 中
- 高
- 爆发中
- 无法判断

### 5. 账户类别

判断发布该内容的账户属于哪一类主体。

说明：
- 这是任务书明确要求保留的维度。
- 但需要明确：这里标注的是**当前可见信息下的账户类别判断**，不是最终身份认证结果。
- 如果输入只有正文，没有账号资料，也应允许输出“未知”。

建议类别：
- 官方机构账号
- 媒体账号
- 记者 / 主持人 / 评论员账号
- 普通个人账号
- KOL / 博主账号
- 组织账号
- Bot / 疑似自动化账号
- 未知

建议证据来源：
- 账号昵称
- 简介
- 认证信息
- 内容口吻
- 持续角色指示语

### 6. 语言风格

判断中性、攻击、讽刺、煽动、情绪化或理性分析等表达方式。

说明：
- 这是典型的内容级可观测维度，应保留。
- 该维度对识别宣传、动员、操纵表达非常重要。

建议标签：
- 中性陈述
- 理性分析
- 评论批评
- 情绪化表达
- 攻击性表达
- 讽刺 / 反讽
- 口号化表达
- 煽动性表达
- 混合

### 7. 内容目的

判断信息分享、观点表达、宣传、动员、攻击、辟谣等传播目的。

说明：
- 该维度比“对象级立场”更适合 T1。
- 它能直接服务 T2 关系抽取和 T6 重点目标识别。

建议标签：
- 信息分享
- 观点表达
- 宣传
- 动员
- 攻击 / 抹黑
- 辟谣 / 澄清
- 质疑 / 追问
- 讽刺 / 嘲讽
- 混合

### 8. 风险等级

判断误导、谣言、对立放大、恐慌、动员和 AIGC 欺骗等风险。

说明：
- 这里建议把“风险等级”定义为综合维度，而不是只看真假。
- 它适合做单条内容级预警。

建议风险来源：
- 明显误导
- 谣言嫌疑
- 对立放大
- 恐慌渲染
- 动员升级
- AIGC 欺骗表达

建议输出形式：
- 总体风险等级：低 / 中 / 高
- 风险类型列表
- 触发证据

### 9. 事件类型 / 议题归属

判断内容主要涉及哪类事件或议题。

说明：
- 这个维度虽然不在你列出的 9 条讨论表述中，但建议保留，因为它对 T2/T3/T4 都很有用。
- 它是对内容对象的分类，不是对内容立场的判断。

建议类别：
- 军事冲突
- 外交事件
- 制裁行动
- 选举政治
- 抗议活动
- 舆论争议
- 信息战 / 传播行动
- 能源安全
- 社会议题
- 其他

### 10. BEND 叙事操纵手法

判断内容是否体现 BEND 叙事操纵手法。

说明：
- 这是本次新增维度。
- 它直接为 T6 重点目标识别提供内容级特征。
- 一条内容允许命中多个手法，也允许不命中。

建议标注以下 8 类：

- 参与（Engage）
  提出与用户有关的重要话题

- 解释（Explain）
  详细阐述主题

- 激发（Excite）
  引发喜悦、兴奋等积极情绪

- 增强（Enhance）
  鼓励继续讨论该话题

- 否定（Dismiss）
  否定该话题的重要性

- 歪曲（Distort）
  歪曲话题的主要内涵

- 泄气（Dismay）
  引发悲伤、愤怒等负面情绪

- 分散（Distract）
  讨论毫不相关的话题

建议输出形式：
- 手法名称
- 置信度
- 对应证据片段
- 判定理由

## 四、建议删除或下沉的维度

以下两项建议不再作为 T1 的正式标注维度。

### 1. 实体 / 对象级立场

原定义：
判断“谁对谁、什么事件或什么议题持有什么立场”。

建议删除原因：
- 这不是单条内容的简单属性，而是一个关系判断。
- 单条内容经常只有局部表达、引用表达、反讽表达，难以稳定判断“谁对谁”的明确立场。
- 它天然更适合放到 T2 做“关系抽取”，而不是 T1 做“内容标注”。

替代建议：
- T1 保留“内容级核心立场”
- T2 再抽“主体 A 对主体 B / 事件 B 的支持、反对、影响等关系”

### 2. 民众态度

原定义：
判断公众、网民或相关群体的支持、批评、担忧、不满等态度。

建议删除原因：
- 这本质上是群体层结论，通常必须结合评论、回复、转发和时间分布来判断。
- 当前评论和回复也是独立输入，只带 `root_id` 指向原始贴文，因此单条内容本身不能代表“民众态度”。
- 如果在 T1 对单条内容直接标民众态度，容易把作者态度误标成公众态度。

替代建议：
- 单条评论 / 回复仍可标其自身的内容级核心立场、观点情绪、语言风格、内容目的、BEND 手法
- “民众态度”下沉到后续聚合模块

---

## 五、建议的讨论版维度清单

为了便于开会讨论，建议最终用下面这组中文维度来表述 T1：

1. 意识形态
2. 内容级核心立场
3. 观点情绪
4. 事件热度
5. 账户类别
6. 语言风格
7. 内容目的
8. 风险等级
9. 事件类型 / 议题归属
10. BEND 叙事操纵手法

删除项：
- 实体 / 对象级立场
- 民众态度

---

## 六、对后续模块的影响

### 对 T1

- 保住任务书要求的核心维度
- 去掉不适合单条内容判断的维度
- 新增 BEND，为 T6 提供更直接输入

### 对 T2

- “对象级立场”从 T1 删除后，可避免 T1 和 T2 在关系层重复定义
- T2 可以更清楚地承担“谁对谁持何立场”的正式抽取职责

### 对 T6

- 可直接使用 T1 的 BEND 标注结果
- “账户类别 + 内容目的 + 语言风格 + 风险等级 + BEND”会形成更好的目标识别特征组合

---

## 七、最终建议

本次建议不是削弱 T1，而是让 T1 回到更适合它的位置：

> **对单条内容做稳定、可证据化、可复用的表达级标注**

据此，建议：

- 保留：事件热度、账户类别、内容级核心立场等任务书硬要求维度
- 删除：实体 / 对象级立场、民众态度
- 新增：BEND 叙事操纵手法
- 统一采用中文讨论维度表述，而不是先用接口字段名组织文档
