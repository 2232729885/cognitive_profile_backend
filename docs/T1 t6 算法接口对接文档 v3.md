## 0. 调用约定

### URL 拼接规则

```
完整请求 URL = base_url + "/" + action
```

`base_url` 由算法组部署后提供，填入 `sub_agent_registry.base_url` 字段。

示例：`base_url = http://algo-t1:18001`，`action = annotate_text`
→ 实际请求：`POST http://algo-t1:18001/annotate_text`

### 通用要求

- 请求方式：统一 `POST`，`Content-Type: application/json`
- 成功响应：HTTP `200`，业务失败也返回 `200`（通过响应体字段表达）
- 超时：T1-T4/T6 默认 30 秒，T3/T5 默认 60 秒（可在 `sub_agent_registry.timeout_seconds` 配置）
- 认证：内网部署，当前无认证要求

### 数据库连接（T5 自行查询）

| 服务 | 地址 | 账密 |
|---|---|---|
| PostgreSQL | `172.16.40.232:5432` | `postgres/postgres`，库名 `postgres` |
| Neo4j | `bolt://172.16.40.232:7687` | `neo4j/neo4jpasswd` |

## 1. T1 自动标注

**职责**：对文本进行多维度自动标注，输出供 T2 使用的结构化提示。
**触发时机**：每条 `social_content`/`news_article` 入库后流水线第一步。
**action**：`annotate_text`

### 请求体

```json
{
  "text": "外部势力被指干预选举进程，多名分析人士对此发表质疑。",
  "language": "zh",
  "annotationTypes": [
    "topics", "keywords", "summary", "language_style",
    "sentiment", "event_type", "content_purpose",
    "aigc_suspicion", "entities_hint"
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| text | String | 是 | 待标注文本 |
| language | String | 否 | 语言代码，默认 zh |
| annotationTypes | String[] | 否 | 标注类型，不传则默认全部 9 项 |

### 响应体

```json
{
```

**关键字段说明：**

| 字段 | 说明 |
|---|---|
| `annotations.aigcSuspicion` | 字符串枚举：`none`/`low`/`medium`/`high`。后端会转换成数值（none→0.0, low→0.2, medium→0.5, high→0.85）写入 `media_contents.aigc_score` |

## 3. T3 实体归一（后台任务，不在实时流水线）

> **架构说明**：T3 不在实时流水线里调用。`EntityDeduplicationJob` 每小时运行一次，精确名称匹配完成后调用 T3 做跨语言模糊归一。T3 只负责判断，后端负责执行 PG 和 Neo4j 的合并操作。

**action**：`resolve_entities`

### 请求体

```json
{
  "entities": [
    {
      "id": "8f54f8d6-21d8-4945-9cd8-ca8daffcfa83",
      "entityType": "person",
      "canonicalName": "Leila Farzan",
      "aliases": ["L. Farzan", "莱拉·法尔赞"],
      "importanceScore": 88.0,
      "sourceIdentifiers": {
        "x": "rw_presstv_b56ecf2b",
        "telegram": "12345678"
      }
    },
    {
      "id": "cb14f38d-48b2-4826-9159-f8f355c84ef4",
      "entityType": "person",
      "canonicalName": "ليلى فرزان",
      "aliases": [],
      "importanceScore": 76.0,
      "sourceIdentifiers": {}
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| entities | Array | 是 | 同一类型的待消歧实体，每次调用只传一种类型 |
| entities[].id | String | 是 | PG 精简实体表的 UUID |
| entities[].entityType | String | 是 | `person`/`organization`/`event`/`narrative` |
| entities[].canonicalName | String | 是 | 标准化名称 |
| entities[].aliases | String[] | 否 | 别名列表，供跨语言匹配参考 |
| entities[].importanceScore | Number | 否 | 0-100，用于确定 survivor（保留分数最高的）|
| entities[].sourceIdentifiers | Object | 否 | 平台原生标识符，key=平台名，value=platform_user_id，person 类型会填充 |

### 响应体

```json
{
  "mergeGroups": [
    {
      "survivorId": "8f54f8d6-21d8-4945-9cd8-ca8daffcfa83",
      "mergedIds": ["cb14f38d-48b2-4826-9159-f8f355c84ef4"],
      "confidence": 0.92,
      "matchMethod": "cross_language"
    }
  ],
  "disjointPairs": [],
  "uncertain": []
}
```

| 字段 | 说明 |
|---|---|
| `mergeGroups[].survivorId` | 归一后保留的实体 ID，必须是请求里存在的某个 id |
| `mergeGroups[].confidence` | **后端只执行 confidence ≥ 0.8 的归一建议**，低于阈值跳过 |
| `mergeGroups[].matchMethod` | `exact_name`/`alias_match`/`cross_language`/`semantic_similarity`/`identifier_match` |
| `uncertain` | 置信度不足、无法判断的实体 ID 列表 |

## 4. T4 向量化

**职责**：将文本或图像转换为向量，后端写入 Milvus。T4 只生成向量，不写 Milvus。
**模型**：`Qwen3-VL-Embedding-8B`（`http://172.16.40.232:18013`）

### action 一：文本向量化

**action**：`generate_text_embedding`

```json
// 请求
{"text": "外部势力被指干预选举进程。", "imageUrl": null}

// 响应
{"embedding": [0.123, -0.456, ...], "modelVersion": "qwen3-vl-embedding-8b-v1.0"}
```

### action 二：图像向量化

**action**：`generate_image_embedding`

```json
// 请求
{"text": null, "imageUrl": "http://172.16.40.232:9000/media-assets/test/image1.jpg"}

// 响应（结构同上）
{"embedding": [...], "modelVersion": "qwen3-vl-embedding-8b-v1.0"}
```

**关键约束**：`embedding` 必须是 **4096 维**，Milvus Collection 的 `dim` 固定为 4096，维度不匹配会写入失败。

- 后端持久化：T00 不写 `identification_results` 表；T01-T10 和 T?? 写库
