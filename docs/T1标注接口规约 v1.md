# T1 自动标注接口规约 v1.1
# 课题四 × T1 算法组接口对接文档

> 生效范围：T1 文本标注、图像标注，课题四后端消费标注结果写入 PG 和 Neo4j
> 对齐来源：RZDK `src/agents/t1_annotation/agent.py`、`T1_标注属性.md`
> 变更说明：v1.1 将 10 个标注维度全部对应到接口字段，补充意识形态/内容级核心立场/事件热度/账户类别/BEND 字段，扩展 sentiment/language_style/risk 结构

---

## 一、接口总览

| 接口 | 说明 | 调用时机 |
|---|---|---|
| `POST /t1/annotate_text` | 文本标注（10 个维度）| T1 流水线阶段，每条 social_content/news_article |
| `POST /t1/annotate_image` | 图像标注（3 个维度）| T1 图像阶段，每个 media_asset |
| `POST /t1/batch_annotate` | 批量文本标注（最多 100 条）| 可选，批量处理场景 |

**请求体：**

```json

  "language": "zh",
  "annotation_types": [
    "topics",
    "keywords",
    "summary",
    "sentiment",
    "language_style",
    "event_type",
    "content_purpose",
    "risk",
    "entities_hint",
    "ideology",
    "overall_stance",
    "event_heat",
    "account_type_hint",
    "bend_tactics"
  ],
  "context": {
    "doc_id": "mc_f480e382-a59b-43cd-b42d-17ad6901e4c9",
    "platform": "x",
    "content_type": "post",
    "author_handle": "ft_user_xxx",
    "published_at": "2026-07-07T10:00:00+08:00",
    "hashtags": ["美伊冲突", "紧急"],
    "parent_content_id": null

}
```

**请求字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `text` | string | **是** | 待标注文本 |

| `annotation_types` | string[] | 否 | 指定标注维度，默认全部 9 个。可按需裁剪 |
| `context` | object | 否 | 内容上下文，辅助提升标注质量 |
| `context.doc_id` | string | 否 | 内容 ID，用于日志追踪 |
| `context.platform` | string | 否 | 来源平台，辅助账户类别判断 |
| `context.content_type` | string | 否 | 内容类型：`post`/`comment`/`reply`/`article` |
| `context.hashtags` | string[] | 否 | 话题标签，辅助话题判断 |

### 2.1 annotations 字段详细说明

> 对应 `T1_标注属性.md` 的 10 个标注维度

---

#### ① topics（议题归属）→ 维度9：事件类型/议题归属（部分）

- 类型：`string[]`
- 说明：内容涉及的议题领域，允许多个
- 建议枚举值：`military`/`geopolitics`/`energy`/`diplomacy`/`election`/`protest`/`sanctions`/`cybersecurity`/`social`/`economics`/`unknown`

#### ② keywords（关键词）→ 辅助维度

- 类型：`string[]`
- 说明：T1 抽取/生成的关键词列表，用于检索和内容理解，最多 10 个

#### ③ summary（一句话摘要）→ 辅助维度

- 类型：`string`
- 说明：T1 生成的简短摘要，不超过 100 字，不是原文截取

#### ④ sentiment（观点情绪）→ 维度3

- 类型：`object`
- 说明：情感极性、主要情绪类型和情绪强度（v1.1 扩展，补全维度3）

```json
{
  "label": "negative",
  "score": 0.88,
  "primary_emotion": "fear",
  "emotion_polarity": "negative",
  "emotion_intensity": "high"
}
```

| 子字段 | 取值 | 说明 |
|---|---|---|
| `label` | `positive`/`negative`/`neutral`/`mixed` | 情感极性 |
| `score` | 0.0-1.0 | 极性强度 |
| `primary_emotion` | `anger`/`fear`/`sadness`/`joy`/`pride`/`anxiety`/`calm`/`contempt`/`null` | 主要情绪类型 |
| `emotion_polarity` | `positive`/`negative`/`neutral`/`mixed` | 情绪极性（与 label 对应）|
| `emotion_intensity` | `low`/`medium`/`high` | 情绪强度 |

#### ⑤ language_style（语言风格）→ 维度6

- 类型：`object`
- 说明：语言表达方式分析（v1.1 扩展，补全维度6）

```json
{
  "formality": "informal",
  "emotional_intensity": "high",
  "style_tags": ["sensational", "mobilization", "panic_inducing"]
}
```

| 子字段 | 取值 | 说明 |
|---|---|---|
| `formality` | `formal`/`informal`/`mixed`/`unknown` | 正式程度 |
| `emotional_intensity` | `none`/`low`/`medium`/`high`/`unclear` | 整体情绪强度 |
| `style_tags` | string[] | 风格标签（多选）：`neutral`/`rational`/`critical`/`emotional`/`aggressive`/`sarcastic`/`slogan`/`sensational`/`mobilization`/`panic_inducing` |

#### ⑥ event_type（事件类型/议题归属）→ 维度9

- 类型：`string`
- 说明：内容涉及的具体事件类型，在 topics 之上细化
- 建议取值：`military_conflict`/`diplomatic_event`/`sanctions`/`election`/`protest`/`cyber_operation`/`regional_security_issue`/`energy_security`/`information_warfare`/`unknown`

#### ⑦ content_purpose（内容目的）→ 维度7

- 类型：`string`
- 说明：判断信息传播的目的
- 取值：`information_sharing`/`opinion_expression`/`propaganda`/`mobilization`/`attack`/`debunking`/`criticism`/`explanation`/`report`/`unknown`

#### ⑧ risk（风险等级）→ 维度8

> v1.1 新增，替代原 `aigc_suspicion`（AIGC 嫌疑合并进 `risk.types`）

- 类型：`object`
- 说明：综合风险等级，涵盖多种风险类型

```json
{
  "level": "high",
  "types": ["panic_inducing", "misinformation", "mobilization"],
  "evidence": "号召囤油、声称战争不可避免",
  "aigc_suspicion": "low"
}
```

| 子字段 | 取值 | 说明 |
|---|---|---|
| `level` | `low`/`medium`/`high` | 综合风险等级 |
| `types` | string[] | 风险类型（多选）：`misinformation`/`rumor`/`polarization`/`panic_inducing`/`mobilization`/`aigc_deception` |
| `evidence` | string | 触发风险的原文证据 |
| `aigc_suspicion` | `none`/`low`/`medium`/`high` | AIGC 嫌疑（合并到 risk 里）|

#### ⑨ entities_hint（实体提示）→ 辅助维度（为 T2 服务）

（字段说明见下方 2.2 节，与 v1.0 一致）

#### ⑩ ideology（意识形态）→ 维度1（v1.1 新增）

- 类型：`object` 或 `null`
- 说明：内容体现的政治倾向和意识形态取向，无明显倾向时返回 `null`

```json
{
  "label": "anti_west",
  "intensity": "high",
  "evidence": "美伊冲突、所有油轮都会被封锁"
}
```

| 子字段 | 取值 | 说明 |
|---|---|---|
| `label` | `nationalist`/`conservative`/`liberal`/`religious`/`anti_establishment`/`pro_west`/`anti_west`/`neutral`/`unclear` | 意识形态标签 |
| `intensity` | `weak`/`moderate`/`strong` | 倾向强度 |
| `evidence` | string | 支撑证据片段 |

#### ⑪ overall_stance（内容级核心立场）→ 维度2（v1.1 新增）

- 类型：`string`
- 说明：整条内容总体立场方向，判断的是内容整体表达层面
- 取值：`support`/`oppose`/`neutral`/`mixed`/`unknown`

#### ⑫ event_heat（事件热度）→ 维度4（v1.1 新增）

- 类型：`string`
- 说明：基于当前这条内容所体现出的热度信号（感知标注，不是全网统计）
- 取值：`low`/`medium`/`high`/`breaking`/`unknown`
- 判断依据：是否使用"刷屏""全网关注""持续发酵"等热度措辞，是否带有明显事件升级感

#### ⑬ account_type_hint（账户类别）→ 维度5（v1.1 新增）

- 类型：`string`
- 说明：基于当前可见信息对发布账户类型的判断（非最终认证结果）
- 取值：`official`/`media`/`journalist`/`individual`/`kol`/`organization`/`bot`/`unknown`
- 判断依据：账号昵称、简介、认证信息、内容口吻

#### ⑭ bend_tactics（BEND 叙事操纵手法）→ 维度10（v1.1 新增）

- 类型：`object[]` 或 `[]`
- 说明：判断内容是否体现 BEND 叙事操纵手法，允许命中多个，未命中返回空数组

```json
[
  {
    "tactic": "Excite",
    "confidence": 0.88,
    "evidence": "突发！海湾战争已经不可避免",
    "reason": "使用感叹号和紧迫性语言激发恐慌情绪"
  }
]
```

| 子字段 | 取值 | 说明 |
|---|---|---|
| `tactic` | 见下表 | BEND 8 类手法之一 |
| `confidence` | 0.0-1.0 | 置信度 |
| `evidence` | string | 原文证据片段 |
| `reason` | string | 判定理由 |

**BEND 8 类手法：**

| 手法 | 英文 | 说明 |
|---|---|---|
| 参与 | `Engage` | 提出与用户有关的重要话题 |
| 解释 | `Explain` | 详细阐述主题 |
| 激发 | `Excite` | 引发喜悦、兴奋等积极情绪 |
| 增强 | `Enhance` | 鼓励继续讨论该话题 |
| 否定 | `Dismiss` | 否定该话题的重要性 |
| 歪曲 | `Distort` | 歪曲话题的主要内涵 |
| 泄气 | `Dismay` | 引发悲伤、愤怒等负面情绪 |
| 分散 | `Distract` | 讨论毫不相关的话题 |

#### entities_hint（实体提示）

- 类型：`object[]`
- 说明：为 T2 信息抽取提供实体线索，每个实体一条

```json
[
  {
    "text": "海湾战争",
    "type_hint": "event",
    "span": {"start": 3, "end": 7},
    "stance": "support",
    "emotion_expression": "worry",
    "emotion_intensity": "high",
    "evidence_ids": ["ev_001"]
  }
]
```

| 子字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `text` | string | **是** | 实体表层文字，**必须是原文的子串** |
| `type_hint` | string | **是** | 实体类型提示：`person`/`organization`/`event`/`location`/`social_account`/`media_content`/`narrative`/`other`/`unknown` |
| `span` | object | 否 | 字符偏移 `{"start": N, "end": N}`，由 T1 本地计算 |
| `stance` | string | 否 | 内容对该实体的立场：`support`/`oppose`/`neutral`/`null` |
| `emotion_expression` | string | 否 | 情绪表达：`worry`/`anger`/`joy`/`fear`/`pride`/`null` |
| `emotion_intensity` | string | 否 | 情绪强度：`low`/`medium`/`high`/`null` |
| `evidence_ids` | string[] | **是** | 对应 `evidence_clues` 里的 `evidence_id`，用于闭环追溯 |

---

### 2.2 evidence_clues 字段说明

- 类型：`object[]` 或 `null`
- 说明：由 `entities_hint` 反推的证据线索池，无实体时为 `null`
- 每条证据和 `entities_hint[i].evidence_ids[0]` 严格对应

```json
{
  "evidence_id": "ev_001",
  "evidence_type": "text_span",
  "raw_content": "海湾战争",
  "span": {"start": 3, "end": 7},
  "confidence": 0.92
}
```

| 字段 | 说明 |
|---|---|
| `evidence_id` | 证据 ID，与 `entities_hint[i].evidence_ids[0]` 一一对应 |
| `evidence_type` | 固定为 `text_span` |
| `raw_content` | 证据原文片段 |
| `span` | 字符偏移 |
| `confidence` | 证据置信度 |

---

### 2.3 quality_control 字段说明

| 字段 | 说明 |
|---|---|
| `auto_label_status` | `success`（全字段LLM成功）/`partial`（部分回落占位）/`failed` |
| `need_human_review` | 是否需要人工审核 |
| `review_reason` | 需要审核的原因，不需要时为空字符串 |
| `schema_version` | 固定为 `t1_annotation_v0.3` |
| `model_version` | 实际使用的模型版本，如 `qwen3:32b` |

### 3.1 图像标注字段说明

#### objects（目标检测）

- 类型：`object[]`
- 每个对象包含：`label`（标签）、`confidence`（置信度 0-1）、`bbox`（边界框 `[x, y, w, h]`）

#### scene（场景分类）

- 类型：`string`
- 图像整体场景标签，如 `maritime`/`military`/`protest`/`meeting`/`outdoor`/`indoor`/`unknown`

#### text_ocr（OCR 文字）

- 类型：`string` 或 `null`
- 图像中识别出的文字，无文字时为 `null`
